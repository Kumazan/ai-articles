## OpenAI 宣布收購 Astral，將 uv、Ruff 等 Python 開源工具納入 Codex 生態

今天，OpenAI 宣布將收購 [Astral](https://astral.sh/)，把這家備受 Python 社群推崇的開源工具公司引入 Codex 生態系。

Astral 開發了一些目前最廣泛使用的 Python 開源工具，讓開發者能以更現代化的工具鏈加速工作流程：

- **uv** — 快速的 Python 依賴與環境管理工具
- **Ruff** — 極速 Python 程式碼風格檢查與格式化工具
- **ty** — 協助在程式碼庫中強制型別安全的工具

這些工具已成為現代 Python 開發的基石，廣泛應用於數百萬名開發者的日常工作流程。

### 為什麼 Codex 需要 Astral？

OpenAI 的 Codex 近年來快速成長：自 2026 年初以來，用戶數成長了 3 倍，使用量更提升了 5 倍，目前每週活躍用戶已超過 200 萬人。

Codex 的核心目標是超越單純的「生成程式碼」，走向能夠參與整個開發工作流的 AI 系統——協助規劃變更、修改程式碼庫、執行工具、驗證結果，並隨時間維護軟體。

Astral 的開發者工具正好位於這個工作流的核心。收購完成後，OpenAI 計畫將這些工具系統整合進 Codex，讓 AI 代理能更直接地與開發者每天依賴的工具協作。

### 開源承諾不變

Astral 創辦人兼 CEO Charlie Marsh 表示：「Astral 一直專注於打造能改變開發者使用 Python 方式的工具——幫助他們更快速地交付更好的軟體。作為 Codex 的一部分，我們將持續演進開源工具，推動軟體開發的前沿發展。」

依據計畫，收購完成後 OpenAI 將繼續支持 Astral 的開源專案——這對倚賴 uv 和 Ruff 的廣大 Python 社群而言是個正面訊號。

Codex 負責人 Thibault Sottiaux 則說：「Astral 的工具被數百萬 Python 開發者使用。透過將他們的專業知識與生態系引入 OpenAI，我們正在加速 Codex 成為能夠跨越整個軟體開發生命週期工作的最強代理的願景。」

### 意義何在？

這次收購清晰地傳達了一個訊號：OpenAI 認為未來的 AI Coding Agent，必須能夠深入整合既有的開發工具鏈，而不只是「補全下一行程式碼」。

從寫程式、到管理依賴、到執行型別檢查、到驗證成果——Codex 正在尋求成為真正的開發夥伴，而 Astral 的工具正是拼圖中缺少的一塊。

收購完成仍需等待慣例的監管審查。在正式交割前，OpenAI 與 Astral 仍為各自獨立運作的公司。
