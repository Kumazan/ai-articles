## 摘要

- OpenAI 認為，企業 AI 已經離開「試用看看」階段，現在開始真正做事了
- 他們提到，企業已經占 OpenAI 營收超過 40%，而且有機會在 2026 年底追上消費端
- Codex 的每週活躍用戶已經超過 300 萬，顯示 agentic workflow 正在快速落地
- OpenAI 想把 Frontier 做成企業內部的統一智能層，讓 agent 能跨系統、跨工具工作
- 另一條主線是 AI superapp，把 ChatGPT、Codex、agentic browsing 和企業流程收進同一個介面
- 文中最重要的訊號，是企業開始從「用 AI 幫忙」轉向「讓 agent 代做事情」
- 對開發者和企業團隊來說，這代表整個工作入口正在被重新定義

<div class="sep">· · ·</div>

剛結束在 OpenAI 的前 90 天，我有機會和數百位客戶交流。最讓我印象深刻的是，他們展現出的急迫感與準備程度。我的職涯大多在科技與企業轉型的交叉點上，但我從沒見過這麼強烈、而且跨產業都如此一致的信念。這些領導者都知道，AI 是他們這一生最重大的變化，現在他們想問的是，該怎麼把公司真正圍繞它重建起來。

這種信念也反映在 OpenAI 的業務上。建立在我們的消費端優勢之上，企業業務現在已經占總營收超過 40%，而且有望在 2026 年底前和消費端持平。Codex 的每週活躍用戶剛突破 300 萬，API 每分鐘處理超過 150 億 tokens，GPT‑5.4 也正在 agentic workflow 裡創下新的使用高點。我們看到新的客戶包括 Goldman Sachs、Phillips 和 State Farm，也在既有客戶那裡持續成長，像 Cursor、DoorDash、Thermo Fisher 和 LY Corporation。

很明顯，我們已經過了實驗期。AI 現在真的在做工作，因此每家公司都在面對兩個核心問題：

- 要怎麼把最強的 AI 放進整個企業，而不只是單一的 copilot 或 assistant？
- 要怎麼讓 AI 真正進入人們每天的工作，幫他們釋放全部潛力？

這兩個問題會定義未來幾年的企業運作方式與競爭方式，而這正是我們企業策略的方向：一方面把 Frontier 做成企業所有 agent 背後的底層智能層，另一方面打造一個統一的 AI superapp，讓員工在裡面把事情做完。

OpenAI 是少數同時在做完整技術堆疊的公司之一，從基礎設施到模型，再到員工每天使用的介面。我們一直在聽客戶的需求，也正在快速成為 AI 的核心基礎設施，讓全世界各種規模的企業都能開始真正把東西做出來，並且有信心走向未來。

正如我們之前說過的，世界正處在一個 capability overhang 的階段，也就是模型已經能做的事，遠遠超過大多數人與企業實際在使用的程度。我們的承諾，就是把這段落差補起來，讓 frontier intelligence 變得可用、可信，而且真正嵌入工作流程之中。

我最常聽到的一件事是，企業已經受夠了彼此不會溝通、只會製造混亂的 AI point solution。他們想要的是一個統一的 AI 作業層，讓 AI coworkers 以公司脈絡為基礎，連上內部系統、外部資料來源，並且受到正確的權限與控管治理。這就是我們透過 [OpenAI Frontier](https://openai.com/index/introducing-openai-frontier/) 所提供的能力，它正在幫助 Oracle、State Farm 和 Uber 這類客戶把 AI 佈署並管理到整個公司。和只把 agent 綁在單一產品或環境裡的方案不同，Frontier 讓 agent 能跨越公司的系統與資料工作，在工具之間移動，並且隨著時間持續變得更好。

除了是一家研究公司，我們也是一家部署公司。我們把和數百家大型企業合作整合 AI agent 的經驗，轉成一個可擴充的基礎。再加上 Frontier Alliances 夥伴 McKinsey & Company、Boston Consulting Group (BCG)、Accenture 和 Capgemini，以及 AWS、Databricks、Snowflake 等合作夥伴，我們幫企業把 OpenAI 的 intelligence 整合進既有的基礎設施與資料生態系。舉例來說，我們和 AWS 正在打造的 [Stateful Runtime Environment](https://openai.com/index/amazon-partnership)，讓 agent 可以保留 context、記住先前工作，並跨公司的工具與資料運作，對複雜的真實世界情境更有用。

當 AI 擴散到整個公司後，它也必須自然地出現在每個人、每個團隊的日常工作流裡。這就是為什麼我們正在朝一個統一 AI superapp 前進，也就是一個地方，讓員工整天都能和 AI agent 一起完成任務，並且跨既有工具採取行動。這個體驗會把最好的 ChatGPT、Codex、agentic browsing，還有更廣泛的能力整合在一起，讓個人與小團隊能完成更多事情。

最近幾個月，我們看到一個明顯轉變：走在前面的使用者，已經從把 AI 當成任務助手，轉向管理一群 agent 去替自己做事。這個變化是從 Codex 這類 agentic 工具開始的，而 Codex 從今年初以來已經成長超過 5 倍。這也包括像 GitHub、Nextdoor、Notion 和 Wonderful 這些客戶，他們正在打造可以端到端執行工程工作的多 agent 系統。我們也開始看到各個職能的員工都把 agent 納入日常流程。舉例來說，我們的銷售團隊會用 agent 研究 inbound prospects、依照 rubric 打分、寄出個人化 email 給合格名單，然後自動更新 CRM。

我們很期待推出更多能讓企業更容易使用 agent 的新方案。OpenAI 最大的優勢之一，就是我們能同時連接個人與工作場景。ChatGPT 每週有 9 億使用者，代表員工其實早就知道怎麼跟它工作。對企業來說，這大幅降低了導入阻力，也加快了員工把瑣碎工作交出去、去做更有野心任務的速度。

這次和客戶交流的經驗，讓我更確信一件事，AI 轉型正在比多數人想像得更快發生。企業需要的是一個真正懂得這場轉變規模、而且能協助他們穩定前進的夥伴。這意味著要走進他們已經在使用的系統，提供從試驗到部署的實際路徑，並且用合適的定價和包裝降低採用門檻。更重要的是，他們需要相信，幫助他們完成這場轉型的公司，真的把他們的成功放在心上，也真的在為他們的需求打造產品。

在 OpenAI，每個部門、每個角色都能感受到這份承諾。我們全力聚焦在持續贏得企業的信任，幫助它們和背後的人，帶著清晰、信心與信任，重新打造面向 AGI 的未來公司。

## 延伸評論｜企業 AI 正在從工具變成基礎設施

這篇文章最有份量的地方，不是又一次宣傳模型多強，而是把企業 AI 的競爭焦點直接推到「工作入口」與「組織作業層」。當 agent 開始接手研究、回信、更新 CRM、追蹤任務，真正重要的就不再只是單次回答品質，而是權限、上下文、可觀測性與跨系統協作能力。

對開發者來說，這代表接下來的重點會從「做一個會聊的 AI」轉向「做一個能穩定跑流程的 AI 系統」。誰能把模型、工具、記憶、sandbox、審核與介面串成一個可長跑的整體，誰就更接近下一代企業軟體的核心位置。
