# AI 文章品質評分器

評估 ai-articles 中的文章（或選文候選），輸出結構化評分表，找出不符標準的文章。

## 使用方式

- **審查單篇文章（路徑或 URL）**：`/ai-articles-quality-scorer <path_or_url>`
- **審查全部已發布文章**：`/ai-articles-quality-scorer all`
- **評估選文候選（URL）**：`/ai-articles-quality-scorer candidate <url>`

## 執行流程

### 1. 取得文章清單

若是 `all` 模式：
- 讀取 `/Users/kumax/.openclaw/workspace/ai-articles/index.md`
- 對每篇文章（`.md` 路徑）執行以下評分流程

若是單篇或候選模式：
- 讀取指定檔案或用 defuddle 抓取 URL

### 2. 五維評分（每維 1-10 分）

**維度一：來源可信度（Source）**

> ⚠️ **重要修正**：這個維度衡量的是「這篇文章內容是否值得相信」，而不是「它來自哪裡」。推文可以給 10 分，官方部落格也可以給 2 分。重點是**內容量**、不是**發布平台**。

| 分數 | 標準 |
|------|------|
| 9-10 | 來源明確、內容具體、有數據或有名字可查證 |
| 7-8 | 有名字的作者、有脈絡的論述、非公告性質 |
| 5-6 | 來源可查，但內容泛泛、缺乏具體細節 |
| 3-4 | 來源模糊（匿名、內容農場），但標題看起來像資訊 |
| 1-2 | 來源不明或造假、純情緒發文、標題與內容不符 |

**評分範例**：
- Ethan Mollick（知名研究者部落格）→ 8
- Simon Willison（頂級獨立 blogger）→ 8
- trq212 X/Twitter thread，具體好用 → **8**（不是 1-2！）
- RenovateQR 內容農場文（有名字但無內容）→ 3-4
- 官方公告，無數據無分析 → 4-5

**維度二：受眾廣度（Reach）**

| 分數 | 標準 |
|------|------|
| 9-10 | 影響所有 AI 使用者或從業者（例：重大模型發布、AI 安全政策）|
| 7-8 | 影響大多數 AI 開發者（例：新的 coding agent 技術、重要 API 變化）|
| 5-6 | 特定族群（例：LLM 推理優化工程師、特定框架用戶）|
| 3-4 | 高度利基（例：特定小工具的使用者）|
| 1-2 | 只有作者或極少數人關心（例：某工具的 workflow 比較）|

**維度三：資訊密度（Depth）**

| 分數 | 標準 |
|------|------|
| 9-10 | 充滿具體數據、案例、可行洞察；讀完有明確收穫 |
| 7-8 | 有論點、有細節，偶有填充 |
| 5-6 | 有基本事實，但缺乏深度分析 |
| 3-4 | 主要是摘要或轉述，無原創觀點 |
| 1-2 | 空洞、純公告、無資訊量 |

**維度四：AI 核心相關性（Relevance）**

| 分數 | 標準 |
|------|------|
| 9-10 | AI/ML 是主題本身（模型、訓練、agent、評估）|
| 7-8 | AI 工具或 AI 產業（coding agent、AI 公司動態）|
| 5-6 | AI 相鄰（AI 政策、AI 投資、AI 倫理）|
| 3-4 | 技術文章，AI 是配角但仍值得翻（安全事件中 AI 受害、AI 被用來攻擊等）|
| 1-2 | 跟 AI 基本無關 |

**評分範例**：
- supply chain 攻擊：AI 工具本身是受害者 → 5-6（不是 3-4）
- AI 用來分析攻擊：AI 是工具 → 7-8

**維度五：時效與獨特性（Freshness）**

| 分數 | 標準 |
|------|------|
| 9-10 | 48 小時內的重大新聞，或常青型深度指南 |
| 7-8 | 一週內的新鮮分析，或與已發布文章無重疊 |
| 5-6 | 兩週內，有些微重疊但角度不同 |
| 3-4 | 舊新聞，或與已發布文章主題高度重疊 |
| 1-2 | 已翻過、過時，或重複舊事件 |

### 3. 計算總分與判定

**總分 = 五維相加（滿分 50）**

| 總分 | 判定 |
|------|------|
| 40-50 | ✅ **優秀** — 強烈推薦翻譯 |
| 30-39 | 🟡 **良好** — 值得翻譯，視當日量決定 |
| 20-29 | ⚠️ **普通** — 只在當日無更好文章時考慮 |
| < 20 | ❌ **不合格** — 不應翻譯或建議下架 |

### 4. 輸出格式

**單篇模式：**
```
📊 文章品質評分
標題：{title}
路徑：{path}

來源可信度：{score}/10 — {reason}
受眾廣度：{score}/10 — {reason}
資訊密度：{score}/10 — {reason}
AI 核心相關性：{score}/10 — {reason}
時效與獨特性：{score}/10 — {reason}

總分：{total}/50 → {判定}

{如果 < 30，加一行：建議行動：下架 / 不翻 / 重新選文}
```

**全庫審查模式（`all`）：**

輸出一份排序表，顯示所有文章評分，從低到高排列。

重點標示：
- ❌ 列出分數 < 20 的文章（建議下架），含具體理由
- ⚠️ 列出 20-29 的文章（邊緣），供人工決定
- 同時輸出：今日/本週已翻數量是否超過品質門檻（> 3篇/日）

## 備註

- 這個 skill 只評分，不修改任何檔案
- 若要移除文章，需另行從 index.md 刪除連結並 git commit
- 評分帶有主觀成分，最終取捨由 Kuma 決定
