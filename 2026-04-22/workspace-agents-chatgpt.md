## 摘要

- OpenAI 新推出的 Workspace Agents，主打讓團隊把常見流程變成可共享的代理。
- 這些代理跑在雲端，可以持續工作、排程執行，也能在 ChatGPT 或 Slack 裡接續互動。
- 它們不是只會回答問題，而是能讀文件、跑程式、串接工具，並在需要時要求人工審批。
- OpenAI 把它定位成 GPTs 的進化版，重點從「個人使用」轉向「組織內共享流程」。
- 真正值得注意的不是 demo，而是它把權限、記憶、監控與協作一起包進同一套產品。

<div class="sep">· · ·</div>

今天，OpenAI 推出 ChatGPT Workspace Agents。團隊現在可以建立共享代理，讓它們處理複雜任務與長流程工作，並且在組織設定好的權限與控管之下運作。

Workspace Agents 是 GPTs 的進化版。它們由 Codex 驅動，能接手很多人原本就會在工作中做的事：整理報告、撰寫程式、回覆訊息。它們跑在雲端，所以即使人離開電腦，也能持續工作。它們也設計成可以在組織內共享，讓團隊可以在 ChatGPT 或 Slack 裡先建立一次、大家一起用，之後再持續調整。

OpenAI 認為，AI 雖然已經幫很多人把個人工作加速了，但企業裡最重要的流程往往仰賴共享上下文、交接與跨團隊決策。Workspace Agents 的設計就是對準這種場景：它們可以從正確的系統蒐集脈絡、依照團隊流程做事、在需要時請人核准，並且在不同工具之間把工作往前推進。OpenAI 舉的例子是銷售團隊：代理可以從通話筆記與客戶研究中整理資訊、篩選新線索，還能直接在業務的收件匣裡草擬跟進信。

要開始使用，只要在 ChatGPT 側邊欄點選 Agents，描述團隊常做的一個流程，ChatGPT 就會一步一步把它變成代理。Workspace Agents 目前提供給 ChatGPT Business、Enterprise、Edu 與 Teachers 方案作為研究預覽。

GPTs 會先繼續保留，讓團隊可以一邊測試 workspace agents、一邊逐步遷移。之後，OpenAI 也會讓把 GPT 轉成 workspace agent 變得更容易。

ChatGPT 也提供導引式流程，讓團隊把工作拆成步驟、接上正確工具、加入 skills，然後一路測到符合預期為止。

OpenAI 列出的幾個可立即建立的代理包括：

- **Software Reviewer**：審查員工提出的軟體需求，對照核准工具與政策，建議下一步，必要時建立 IT 工單。
- **Product Feedback Router**：監控 Slack、客服頻道與公開論壇，把回饋整理成優先順序明確的工單與每週摘要。
- **Weekly Metrics Reporter**：每週五抓資料、產生圖表、撰寫摘要，並把報告分享給團隊。
- **Lead Outreach Agent**：研究進站名單、依資格標準評分、撰寫個人化跟進信，並更新 CRM。
- **Third-Party Risk Manager**：研究供應商，評估制裁、財務狀況與聲譽風險，產出結構化報告。

也可以直接用模板起步，像是 finance、sales、marketing 等，模板本身就帶有內建 skills 與建議工具，方便快速客製化。

Workspace Agents 可以在多種工具之間蒐集脈絡並採取行動。它們由雲端中的 Codex 驅動，因此擁有檔案、程式碼、工具與記憶組成的工作空間。它們做的不只是回應提示：還能寫程式、執行程式、使用已連接的應用程式、記住學到的內容，並在多步驟任務中持續推進。

它們也能在你不在線上的時候繼續工作。可以把它們設成排程執行，或部署到 Slack 裡，讓它們接住隨時進來的請求。OpenAI 也舉了產品團隊的例子：團隊做了一個代理，會主動在 Slack 頻道回覆員工問題，提供清楚答案、連結相關文件，必要時還能開 ticket。這讓團隊更快解除卡點，也比較不會漏掉重要後續。

目前，團隊可以在 ChatGPT 與 Slack 中與代理互動，之後還會支援更多介面。代理會出現在工作本來就發生的地方，幫助團隊用更少的協調成本把事情推進。

分享之後，團隊也可以從 ChatGPT 側邊欄的 Agents 分頁管理與探索 workspace agents。

OpenAI 強調，很多組織知識其實分散在不同人與不同系統裡。Workspace Agents 提供了一種把這些知識轉成可重複使用工作流程的方法：流程會遵守正確規範、使用正確工具，還能被整個組織共享。

OpenAI 再舉一個會計團隊的例子：代理可以準備月結的關鍵工作，從分錄、資產負債表對帳，到差異分析都包進去。它能在幾分鐘內完成工作，產出附帶底層輸入與控制總數的 workpaper 供人審核，也會遵守內部政策。這個代理可以在 ChatGPT 裡讓團隊任何人使用，也可以加到 Slack 頻道裡，讓大家圍繞輸出一起協作。

因為代理有記憶，而且能在對話中被引導與修正，所以它們會隨著團隊使用而變得更好。時間拉長後，代理也會變成維持團隊知識更新的一種實用方式：建立一次，透過使用持續改進，再分享或複製給新流程使用。

當你把工作交給代理時，控制權仍然在你手上。你可以決定它能用哪些工具與資料、能做哪些動作、哪些步驟必須先批准。像是編輯試算表、寄出 email、加入行事曆事件這些敏感動作，都可以要求代理先徵求許可。

分享出去之後，分析工具也能讓你看到代理的使用情況，包括完成了多少次執行、多少人正在使用它。

Workspace Agents 也具備企業級監控與控管，讓管理者能在保護敏感資料的同時，讓團隊更安全地加快速度。ChatGPT Enterprise 與 Edu 的管理員可以控制不同使用者群組能接觸哪些連接工具與動作，也能管理誰可以使用、建立與分享代理。

OpenAI 也提到內建防護可以幫助代理在遇到誤導性的外部內容時維持對齊，包括 prompt injection 攻擊。Compliance API 則讓管理員能看到每個代理的設定、更新與執行紀錄，方便監控與管控。必要時，管理員也可以停用代理。

之後，管理員還能在後台看見組織內建立的所有代理，包括使用趨勢與連接的資料來源。

早期測試者已經看到較一致的結果，也多出不少時間去做更有價值的工作。OpenAI 引用 Rippling AI Engineering 的 Ankur Bhatt 說法，指出最難的不是模型，而是整合、記憶與使用者體驗；Workspace Agents 把這些工作壓縮掉，讓銷售顧問能端到端做出一個 Sales Opportunity agent，不必再依賴整個工程團隊。

Workspace Agents 目前對 ChatGPT Business、Enterprise、Edu 與 Teachers 方案提供研究預覽；Enterprise 與 Edu 管理員可以用角色權限開啟。

它們會在 2026 年 5 月 6 日前免費，之後開始採用 credit-based pricing。

OpenAI 表示，接下來還會加入更多功能，包括可自動啟動工作的 triggers、更好的儀表板、更多可對 business tools 採取行動的方式，以及對 Codex app 的 Workspace Agents 支援。

團隊做得最好的工作，通常發生在知識更容易找到、流程更容易遵循、也更容易在工作流裡直接得到協助的時候。Workspace Agents 可以算是朝那個方向走的一步：讓 AI 真正待在大家已經在使用的工具和對話裡，幫團隊少花一點協調時間，多一點時間去創造、建造與做決策。

## 延伸評論：把流程代理化，最後拼的其實是組織治理能力

這篇最重要的地方，不是又多了一個「更會做事的 agent」demo，而是 OpenAI 直接把焦點放到組織流程：共享、權限、審批、監控、分析。這表示 agent 競爭已經從單點能力，開始進入誰能把企業工作流真正包進去。

對很多團隊來說，真正的阻力不是模型不夠強，而是流程太散、責任太模糊、工具太多。Workspace Agents 如果真的能把這些拼起來，它賣的就不是聊天體驗，而是把知識工作變成可複用的產品。

但同時，這也把問題推回治理層：誰能建、誰能改、誰能審、誰能停？如果這些答案不夠清楚，代理越強，風險也越大。
